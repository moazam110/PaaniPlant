4. DATABASE DESIGN

üóÑÔ∏è Data Models

Customer Schema

Field Type Required Description
_id ObjectId Auto Unique identifier
name String Yes Customer full name
phone String No Contact number
address String Yes Delivery address
defaultCans Number No Default order quantity
pricePerCan Number Yes Price per water can (0-999)
notes String No Special instructions
createdAt Date Auto Record creation timestamp
updatedAt Date Auto Last modification timestamp

Delivery Request Schema

Field Type Required Description
_id ObjectId Auto Unique identifier
customerId ObjectId Yes Reference to Customer
customerName String Yes Denormalized customer name
address String Yes Delivery address
cans Number Yes Quantity of water cans
priority Enum Yes normal | urgent
status Enum Yes pending | pending_confirmation | processing | delivered | cancelled
requestedAt Date Auto Request creation time
deliveredAt Date No Completion timestamp
internalNotes String No Staff notes

9. TECHNICAL CHALLENGES & SOLUTIONS

üéØ Key Challenges Addressed

Frontend-Backend Integration

Challenge: Connecting deployed frontend to backend API

Solution: Centralized API configuration with environment-based URL management

// src/lib/api.ts export const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL || 'http://localhost:4000'; export const buildApiUrl = (endpoint: string) => `${API_BASE_URL}${endpoint}`;

Mobile Responsiveness

Challenge: Optimizing complex data tables for mobile devices

Solution: Progressive disclosure with responsive column visibility

// Mobile: Customer, Cans, Status, Actions // Tablet: + Address, Priority // Desktop: + Requested timestamp className="hidden sm:table-cell md:table-cell lg:table-cell"

Database Consistency

Challenge: Ensuring data integrity across customer and delivery collections

Solution: MongoDB references with denormalized fields for performance

// Delivery Request Schema customerId: { type: ObjectId, ref: 'Customer' } customerName: { type: String } // Denormalized for faster queries

Real-time Updates

Challenge: Keeping dashboard data synchronized

Solution: Periodic data fetching with health check monitoring

// Auto-refresh every 10 seconds const interval = setInterval(fetchData, 10000);
